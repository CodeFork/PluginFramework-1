PluginFramework Concept

The key players inside the PluginFramework is the AssemblyContainer, the PluginRepository and the PluginCreator.

AssemblyContainer

The AssemblyContainer monitors one or more PluginDirectory and keeps the container up to date with which assemblies that is currently available on disk. It then exploses those assemblies through two interfaces, IAssemblySource and IAssemblyRepository.

PluginRepository

The PluginRepository reads and remembers plugins exposed through registered IPluginSources and allows users to query for plugins that matches a specified filter

PluginCreator

The PluginCreator is an untility class that exposes static functions for finding and creating instances of IPluginCreator inside AppDomains.

PluginDirectory

The PluginDirectory monitors changes to assemblies in a directory on disk. It then raises the events FileFound and FileLost on it's IPluginDirectory interface accordingly.

IAssemblySource

The IAssemblySource exposes changes to the container through the AssemblyAdded and AssemblyRemoved events.

IAssemblyRepository

The IAssemblyRepository allows external code to fetch the raw data for an assembly using the assemblys full name.

IPluginSource

A plugin source exposes changes to plugins through the PluginAdded and PluginRemoved events. The class PluginExtractor implements IPluginSource.

PluginExtractor

The PluginExtractor listens to IAssemblySource events. Searches the assemblies for defined plugins through reflection. Found plugins is exposed through the IPluginSource interface.

PluginFilter

A plugin filter describes what qualities a plugin must fulfil to be returned from a query to an IPluginRepository. Plugin filters is created using static methods on the Plugin class. Plugin filters can then be combined into complex filters using the & and | operators

IPluginCreator

The IPluginCreator interface is used to create plugins inside an AppDomain. The IPluginCreator uses an IAssemblyRepository to fetch and load missing assemblies inside the AppDomain.
